Changelog

Beta Versionen

[V0.2.1 // 17.10.2025]: Quality-of-Life Features & Vorbereitung Impersonation

  • Automatische Anmeldung hinzugefügt: Benutzer werden beim Öffnen der URL automatisch eingeloggt, sofern ein valider und nicht abgelaufener Token besteht.
  • Admins können nun Fachbereiche anlegen, um organisatorische Strukturen abzubilden.
  • Beginn der Implementierung eines Impersonationsmodus: Admins sollen sich in verschiedene Fachbereiche einloggen und Mod-Aktionen ausführen können. Dies dient der vereinfachten Fehlerreplikation und Analyse.
  • Neue Overlay-Art hinzugefügt, die nicht mit Escape geschlossen werden kann. Dieses Overlay zwingt zur Passwortänderung beim ersten Login oder nach Passwort-Reset.
  • Deploy-Vorgang auf den Server verschoben, da Fehler im Code entdeckt wurden, darunter das Blockieren der Enter-Taste beim Login sowie Authentifizierungsprobleme bei zugewiesenen Rollen.

V0.2.1 setzt auf weitere Quality-of-Life-Optimierungen und legt die Grundlage für den Impersonationsmodus. Das System wird nutzerfreundlicher durch automatische Logins bei gültigen Tokens und klarere Sicherheitsmaßnahmen beim Passwortmanagement. Das Deployment verzögert sich jedoch aufgrund letzter Bugfixes.

[V0.2.0 // 09.10.2025]: Backend-Neustrukturierung & Vorbereitung auf Deployment

  • Komplette Backend-Infrastruktur überarbeitet und optimiert, wodurch der Server nun deutlich schneller arbeitet.
  • Alle Routen und Requests laufen nun stabil und ohne Fehler.
  • Umstellung von JSON auf PostgreSQL vollständig abgeschlossen, wodurch viele Logikfehler und Probleme der alten Datenspeicherung behoben wurden.
  • System ist nun theoretisch produktiv einsetzbar, weshalb die Einrichtung eines Servers für das CLOCKWORK-Deployment gestartet wurde.
  • Erste Testphase des produktiven Deployments in der IT-Abteilung geplant.

V0.2.0 markiert den größten infrastrukturellen Sprung bisher. Das Backend wurde vollständig restrukturiert, die Performance gesteigert und die Kompatibilität zwischen Frontend und Datenbank perfektioniert. Damit ist CLOCKWORK erstmals bereit für den produktiven Betrieb.

[V0.1.7 // 17.09.2025]: Verbesserungen am Frontend & Mod-Panel-Erweiterung

  • Frontend weiter überarbeitet und robuster gemacht, zahlreiche kleine Tweaks und Stabilitätsverbesserungen umgesetzt.
  • Navigation verbessert, flüssigere Übergänge und intuitivere Menüstruktur.
  • Mod-Panel erweitert: neben der Planverwaltung können nun auch Mitarbeiter- und Systemnutzer-Daten verwaltet werden.
  • Im Header wird nun die aktuelle Berechtigungsstufe (Admin, Mod) angezeigt - normale User sehen keinen Hinweis.
  • Auf der Anmeldungsseite wurde ein kurzer Hinweis ergänzt, der erklärt, wie man Zugang zum System erhält.
  • Einige Fehlermeldungen wurden sprachlich überarbeitet, um klarer und verständlicher zu sein.

V0.1.7 stärkt die Stabilität und Bedienbarkeit des Frontends. Das Mod-Panel wächst zu einem vollwertigen Verwaltungstool mit Zugriff auf Pläne, Mitarbeiter und Systemnutzer. Zusätzlich wurden neue UI-Hinweise und klarere Fehlermeldungen integriert, um die Benutzererfahrung weiter zu verbessern.

[V0.1.6 // 11.09.2025]: Moderator-Komponente & UX-Verbesserungen

  • Moderator-Komponente hinzugefügt, da die Datenbank-Rollen nun implementiert sind. Die Admin-Komponente wird später migriert.
  • Mod-Guard und Navigation zum Mod-Panel ergänzt, Angular-Route zur Mod-Komponente hinzugefügt.
  • AuthService akzeptiert jetzt die neuen Datentypen vom PostgreSQL-Server für Systemnutzer.
  • UX-Verbesserungen: Login-Frame muss nicht mehr fokussiert sein, Enter löst Anmeldung aus; Overlays können jetzt mit Escape geschlossen werden.
  • UI/UX-Verbesserungen basierend auf Beta-Feedback: keine Platzhalterzahlen mehr in Admin-Komponente, alle Nutzernamen werden vor dem Senden an das Backend in Kleinbuchstaben konvertiert.
  • Kleinere Misc-Updates und Optimierungen.

V0.1.6 bringt die neue Moderator-Komponente mit, reflektiert die Datenbank-Rollen und die damit verbundenen Berechtigungen. UX und UI wurden weiter optimiert: Login kann komfortabler genutzt werden, Overlays lassen sich schnell schließen, und die Datenkonsistenz im Admin-Bereich wurde verbessert. Die Admin-Komponente wird in einem späteren Update vollständig migriert.

Alpha Versionen

[V0.0.4 // 26.08.2025]: UI/UX Optimierungen & JWT-basierte Authentifizierung

  • Die Benutzeroberfläche und Nutzerführung wurden weiter verfeinert, um ein intuitiveres und konsistenteres Erlebnis zu bieten.
  • Der gesamte Authentifizierungsprozess wurde grundlegend überarbeitet: Anstatt von Express-Sessions werden nun JSON Web Tokens (JWT) verwendet, was das System leichter wartbar und skalierbar macht.
  • Die Access-Control-Mechanismen wurden verbessert, sodass Berechtigungen zuverlässiger geprüft und geschützt werden.
  • Frontend und Backend wurden angepasst, um die neue JWT-Authentifizierung nahtlos zu unterstützen.
  • Kleinere Bugfixes und Optimierungen an bestehenden Komponenten zur Erhöhung der Stabilität und Performance.

Mit Version V0.0.4 wurde die Basis für ein moderneres, einfach zu wartendes Authentifizierungssystem gelegt. Gleichzeitig sorgt das UI-/UX-Update für ein angenehmeres Arbeiten und bereitet das Portal auf zukünftige Erweiterungen vor.
